2. BGD和SGD线性回归算法的设计¶

为方便实现线性回归模型，我设计了一个类，能够通过训练集进行训练，其中提供了两种优化方法：批量梯度下降（BGD） 和 随机梯度下降（SGD），并且可对输入X进行预测：$$\hat{y}=X\theta $$,并与此同时可分别记录训练集以及测试集中的MSE误差。

算法BGD介绍：

批量梯度下降（BGD）使用整个训练集（所有样本）来计算损失函数相对于参数 theta 的梯度。它通过以下步骤更新参数：

1. 预测值计算：利用当前的 theta 进行预测，即得到 $$\hat{y} = X \theta$$ 2. 计算梯度：因为 均方误差损失函数为$$J(\theta)=\frac1m\sum(y-\hat{y})^2$$，计算得到关于$X$的梯度为 $$\nabla_\theta J(\theta)=-\frac2mX^T(y-\hat{y})$$，其中 X 是输入特征矩阵，y 是真实目标标签，m 是样本数量。

3. 更新参数：按照 $$\theta = \theta - \alpha \times \nabla_\theta J(\theta)$$ 的公式更新参数，使得损失函数逐渐减小,趋于收敛。

def batch_gradient_descent(self):
    gradients = 1 / self.m * self.X_train.T.dot(self.X_train.dot(self.theta) - self.y_train)
    self.theta -= self.alpha * gradients

算法SGD介绍： 随机梯度下降在每次迭代中，随机选择一个样本来计算梯度并更新参数。这种方法会更频繁地更新 theta，但每次更新的方向并不一定是最优的，因为它只基于单个样本的梯度。

1. 随机选择样本： 每次迭代随机选择一个样本 xi 和对应的标签 yi。

2. 计算梯度： SGD 的关键不同在于，它每次只用一个样本来更新参数，而不是整个训练集。具体来说，第i个样本对应的损失函数为： $$J(\theta;x^{(i)},y^{(i)})=\left(h_\theta(x^{(i)})-y^{(i)}\right)^2$$,由此可求出单个样本对于每个参数的梯度的计算式：$$\nabla_\theta J(\theta;x^{(i)},y^{(i)})=2\left(h_\theta(x^{(i)})-y^{(i)}\right)x^{(i)}$$

3. 更新参数： 按照 $$\theta:=\theta-\alpha\cdot\nabla_\theta J(\theta;x^{(i)},y^{(i)})$$更新 theta。

def stochastic_gradient_descent(self):
    for i in range(self.m):
        random_idx = np.random.randint(self.m)
        xi = self.X_train[random_idx:random_idx + 1]
        yi = self.y_train[random_idx:random_idx + 1]
        gradients = -2 * xi.T.dot(yi - xi.dot(self.theta))
        self.theta -= self.alpha * gradients

init：构造函数初始化了学习率 learning_rate、迭代次数 epochs 以及优化方法 method（'batch' 对应批量梯度下降，'stochastic' 对应随机梯度下降）；

class LinearRegression:
    def __init__(self, learning_rate=0.01, epochs=1000, method='batch'):
        self.alpha = learning_rate
        self.epochs = epochs
        self.method = method  # 'batch' for BGD, 'stochastic' for SGD

    def fit(self, X_train, y_train, X_test, y_test):
        self.m, self.n = X_train.shape
        # 随机初始化回归系数
        self.theta = np.random.randn(self.n)
        self.X_train = X_train
        self.y_train = y_train
        self.X_test = X_test
        self.y_test = y_test

        # 历史记录 MSE
        self.train_history = []
        self.test_history = []

        for epoch in range(self.epochs):
            if self.method == 'batch':
                self.batch_gradient_descent()
            elif self.method == 'stochastic':
                self.stochastic_gradient_descent()

            # 记录每次迭代训练集和测试集的 MSE
            train_mse = self.mean_squared_error(self.predict(self.X_train), self.y_train)
            test_mse = self.mean_squared_error(self.predict(self.X_test), self.y_test)
            self.train_history.append(train_mse)
            self.test_history.append(test_mse)

    def batch_gradient_descent(self):
        # gradients= m/1* XT (Xθ−y)
        gradients = 1 / self.m * self.X_train.T.dot(self.X_train.dot(self.theta) - self.y_train)
        # print(self.X.T.shape)
        # print(self.theta.shape)
        # print(self.y.shape)
        # print(gradients.shape)
        self.theta -= self.alpha * gradients

    def stochastic_gradient_descent(self):
        for i in range(self.m):
            random_idx = np.random.randint(self.m)
            xi = self.X_train[random_idx:random_idx + 1]
            yi = self.y_train[random_idx:random_idx + 1]
            # print(xi.shape)
            # print(yi.shape)
            gradients = -2 * xi.T.dot(yi - xi.dot(self.theta))
            self.theta -= self.alpha * gradients

    def predict(self, X):
        # print(np.dot(X, self.theta))
        return np.dot(X, self.theta)

    def mean_squared_error(self, predictions, y):
        # print(predictions)
        # print(y)
        return np.mean((predictions - y) ** 2)

2.中级要求实现：探究回归模型在机器学习和统计学上的差异。¶

  • 回归模型在机器学习领域和统计学领域中都十分常用，而且使用方法也相似，但其实际的含义具有本质的区别。本部分我将从总结差异、代码实现、综合评价三个方面来讨论此话题。

1. 总结差异¶

对于机器学习而言，主要目标是预测。通过算法对训练数据进行优化，最小化损失函数（如均方误差MSE）。它通常依赖于大规模计算，从而能够处理高维、复杂、非线性数据，但其解释性相对较弱，本质上还是一种输入与输出之间的预测，在执行过程中会更注重性能与预测精度。在执行形式上来说，机器学习采取迭代的方式来逼近最优解。

而对于统计学而言：主要目标是解释。统计方法更注重模型的假设和推断，关注变量之间的关系、模型拟合优度、假设检验、置信区间等。在统计学中，更加注重因果推断，基于假设的线性模型通常有一定的解释性。在执行形式上来说，统计学方法通过严格的数学过程推导，能够静态地一次性地给出最终的参数估计。

2. 统计学方法代码实现¶

对于本问题，我们完全可以用统计学中的最小二乘法来完成多元线性回归，来得到多个自变量（特征）对一个因变量（标签）的影响。其数学关键过程如下： 回归模型表达式：$$y=X\beta+\epsilon $$

  • y 是 m×1 的因变量向量（目标值），
  • X 是 m×n的特征矩阵，
  • β 是 n×1的回归系数向量，
  • ϵ 是 m×1的误差项。
  • 目标是通过最小二乘法估计 β，即最小化以下损失函数： $$\min_\beta\|y-X\beta\|^2=(y-X\beta)^T(y-X\beta)$$ 解这个问题的闭式解为： $$\hat{\beta}=(X^TX)^{-1}X^Ty$$ 其中，$\hat{\beta}$ 是最小二乘法得到的回归系数估计。

# 划分特征和标签
X = data.iloc[:, :-1].values
y = data.iloc[:, -1].values

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, stratify=y, random_state=42)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
# 为了向量相乘方便，在训练集X左侧添加全为1的一列
X_train = np.hstack([np.ones((X_train.shape[0], 1)), X_train])


X_test = scaler.transform(X_test)
# 在测试集X左侧也添加全为1的一列
X_test = np.hstack([np.ones((X_test.shape[0], 1)), X_test])

# OLS最小二乘法的实现
def ordinary_least_squares(X, y):
    # 计算 (X^T X)^(-1) X^T y
    beta = np.linalg.inv(X.T.dot(X)).dot(X.T).dot(y)
    return beta

# 通过OLS获得统计学参数估计
beta_hat = ordinary_least_squares(X_train, y_train)

# 预测函数
def predict(X, beta):
    return np.dot(X, beta)
    
# 在训练集上进行预测
y_train_pred = predict(X_train, beta_hat)
# 在测试集上进行预测
y_test_pred = predict(X_test, beta_hat)

# 计算训练集 MSE
mse_train = np.mean((y_train_pred - y_train) ** 2)
# 计算测试集 MSE
mse_test = np.mean((y_test_pred - y_test) ** 2)

# 输出结果
print(f'Train MSE: {mse_train}')
print(f'Test MSE: {mse_test}')

# 打印回归系数

3. 综合评价¶

可以观察到，对于本数据集来说，使用统计学的方法获得回归模型预测的结果更方便，同时也具有相持平的精度（在相同的数据划分下测试集预测MSE都在0.58左右），但这并不意味着在线性回归模型领域，统计学方法就优于机器学习基于梯度下降的方法。我对此查阅资料，对两者之间有以下几点联系与区别的思考：

统计学方法相对于机器学习的优势：

对于规模不大、且特征数合适的数据集，线性回归具有很高的计算效率和解释性。

统计学方法相对于机器学习的局限性：

  1. 对于数据规模的局限性：

当数据规模很大时（例如数百万个样本和特征），最小二乘法的计算复杂度会变得很高。OLS 需要计算大规模的矩阵的逆，这个操作的时间复杂度为 O(n^3)，当 n 很大时，最小二乘法就不行了；而此时机器学习的梯度下降就具有优势：梯度下降是增量式的优化方法，特别是随机梯度下降（SGD），每次只对单个样本或一小批样本进行更新。相比于 OLS，它不会出现高复杂度，其内存需求更低，适合处理大规模数据集，尤其是在分布式环境下。

  1. 对于数据形状的局限性：

在高维数据（特征数量大于样本数量）情况下，矩阵$ X^TX$可能是奇异的，这将导致 OLS 解存在多重共线性问题，回归系数估计变得不稳定；而我通过查阅资料了解到，机器学习引入了正则化技术，如Lasso和Ridge回归，可以在高维问题中有效防止过拟合。这些正则化方法通过增加惩罚项使得模型更加稳定，而它们通常是通过梯度下降或其他数值方法来求解的。

两种方法执行上的差异：

最小二乘法通常假设数据是静态的，需要在数据集固定后一次性计算出回归系数，不能应对动态更新的过程；而对于机器学习而言，数据是在过程中不断更新的（比如要估计的参数），梯度下降特别是随机梯度下降非常适合在线学习场景。它可以在数据逐渐到达时，实时更新模型，而不必重新处理整个数据集。这在需要连续学习或数据流场景（如推荐系统、实时预测）中特别有用。

3.高级要求实现：实现岭回归算法，求解训练样本的岭回归模型，平均训练误差和平均测试误差¶

对于本题而言，由于之前已经实现了原始线性回归模型的BGD和SGD，因此我将在此基础上修改来实现BGD和SGD方法的岭回归模型。

1. 岭回归模型分析¶

岭回归的损失函数是： $$J(\theta)=\frac1{2m}\sum_{i=1}^m(h_\theta(x^{(i)})-y^{(i)})^2+\lambda\sum_{j=1}^n\theta_j^2$$ 其中：$\lambda$是正则化参数，控制惩罚项的强度。$\theta_j$是回归系数，不包括偏置项$\theta_0$。和第一部分内容中的类似，$$h_\theta ( x^{( i) }) = \theta ^Tx^{( i) }$$ 是线性回归的预测值。

2. 岭回归模型编程实现与结果呈现¶

相对之前的模型，在梯度下降的过程中，只需在计算梯度时加上正则化项即可。具体来说，对于批量梯度下降和随机梯度下降的实现，可以修改梯度计算的部分来实现岭回归。下面是完整代码与介绍（此代码基本和前面相同，只多了一个参数$\lambda $和修改了梯度更新项）：

class Ridgeregression:
    def __init__(self, learning_rate=0.01, epochs=1000, method='batch', lambda_=0.1):
        self.alpha = learning_rate
        self.epochs = epochs
        self.method = method  # 'batch' for BGD, 'stochastic' for SGD
        self.lambda_ = lambda_  # 正则化参数

    def fit(self, X_train, y_train, X_test, y_test):
        self.m, self.n = X_train.shape
        # 随机初始化回归系数
        self.theta = np.random.randn(self.n)
        self.X_train = X_train
        self.y_train = y_train
        self.X_test = X_test
        self.y_test = y_test

        # 历史记录 MSE
        self.train_history = []
        self.test_history = []

        for epoch in range(self.epochs):
            if self.method == 'batch':
                self.batch_gradient_descent()
            elif self.method == 'stochastic':
                self.stochastic_gradient_descent()

            # 记录每次迭代训练集和测试集的 MSE
            train_mse = self.mean_squared_error(self.predict(self.X_train), self.y_train)
            test_mse = self.mean_squared_error(self.predict(self.X_test), self.y_test)
            self.train_history.append(train_mse)
            self.test_history.append(test_mse)

    def batch_gradient_descent(self):
        # 加入正则化的梯度计算: m/1* XT (Xθ−y) + λθ
        gradients = (1 / self.m) * self.X_train.T.dot(self.X_train.dot(self.theta) - self.y_train)
        # 对 theta 的正则化，不对偏置项 (theta_0) 进行正则化
        gradients += self.lambda_ * np.r_[[0], self.theta[1:]]
        self.theta -= self.alpha * gradients

    def stochastic_gradient_descent(self):
        for i in range(self.m):
            random_idx = np.random.randint(self.m)
            xi = self.X_train[random_idx:random_idx + 1]
            yi = self.y_train[random_idx:random_idx + 1]
            gradients = -2 * xi.T.dot(yi - xi.dot(self.theta))
            # 对 theta 的正则化，不对偏置项 (theta_0) 进行正则化
            gradients += self.lambda_ * np.r_[[0], self.theta[1:]]
            self.theta -= self.alpha * gradients

    def predict(self, X):
        return np.dot(X, self.theta)

    def mean_squared_error(self, predictions, y):
        return np.mean((predictions - y) ** 2)
